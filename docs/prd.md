# 需求文档

## 1. 应用概述

### 1.1 应用名称
寻味 - 城市美食探店推荐网站

### 1.2 应用描述
基于已有前端代码（https://github.com/yu081621/Web）改造的全栈美食探店推荐平台，用户可浏览店铺信息、发布探店日记、收藏打卡店铺、关注其他用户；管理员可管理店铺、审核日记、管理用户和评论。

### 1.3 核心目标
- 用户可注册登录、浏览店铺、发布日记、互动交流
- 管理员可管理内容、审核日记、维护平台秩序
- 保持原有暖橙色系简约清新的视觉风格

## 2. 用户与使用场景

### 2.1 目标用户
- **普通用户**：美食爱好者、探店达人、寻找餐厅的消费者
- **管理员**：平台运营人员
- **超级管理员**：平台最高权限管理者

### 2.2 核心使用场景
- 用户搜索并筛选心仪餐厅，查看评分和日记
- 用户到店消费后发布探店日记，分享体验
- 用户收藏想去的店铺，标记已打卡的店铺
- 用户关注其他探店达人，查看其发布的日记
- 管理员审核用户发布的日记，管理店铺和用户
- 超级管理员生成邀请码、升级管理员权限

## 3. 页面结构与功能说明

### 3.1 页面层级结构

```
寻味网站
├── 用户端
│   ├── 首页
│   ├── 探店页
│   ├── 探店日记页
│   ├── 发布日记页
│   ├── 榜单页
│   ├── 店铺详情页
│   ├── 日记详情页
│   ├── 个人中心页
│   └── 登录注册页
└── 管理端
    ├── 管理员登录页（/admin/login）
    └── 管理员后台
```

### 3.2 用户端页面功能

#### 3.2.1 首页
- **Hero搜索区**：输入关键词搜索店铺或日记
- **美食分类导航**：展示8类美食分类（火锅/烧烤/甜品/中餐/西餐/日料/韩料/小吃），点击跳转探店页并应用筛选
- **精选推荐店铺**：展示店铺卡片（店铺名称、类型、商圈、人均价格、综合评分），点击进入店铺详情页
- **热门榜单**：展示热门榜/新店榜/性价比榜三类榜单的前几名，点击查看更多跳转榜单页
- **探店日记预览**：展示最新或热门日记卡片，点击进入日记详情页

#### 3.2.2 探店页
- **搜索框**：输入关键词搜索店铺
- **菜系筛选**：全部/火锅/烧烤/中餐/西餐/日料/韩料/甜品
- **商圈筛选**：全部/春熙路/太古里/IFS/宽窄巷子/九眼桥
- **排序选择**：综合排序/评分最高/人均最低/最新开业
- **店铺卡片列表**：展示店铺名称、类型、商圈、人均价格、综合评分、收藏/打卡按钮
- **收藏/打卡操作**：登录后可标记想去/已打卡/收藏

#### 3.2.3 探店日记页
- **标签过滤**：推荐/最新/热门/关注（关注需登录）
- **地区筛选**：全部/春熙路/太古里/IFS/宽窄巷子/九眼桥
- **日记卡片列表**：展示用户头像、用户名、发布时间、店铺名称、店铺位置、日记图片、日记标题、日记内容摘要、点赞数、评论数
- **关注按钮**：登录后可关注日记作者
- **点击卡片**：进入日记详情页

#### 3.2.4 发布日记页
- **店铺选择（可选）**：输入店铺名称从列表中选择，或选择「不关联店铺」跳过此步骤
- **店铺位置**：选择店铺后自动填充位置，未选择店铺时此项为空
- **总体评分**：星级评分（1-5星）
- **分项评分**：口味/环境/服务/性价比（1-5星）
- **日记标题**：输入标题
- **探店心得**：输入日记内容
- **图片上传**：上传多张图片
- **标签添加**：输入标签（如「环境好」「性价比高」）
- **提交按钮**：提交后日记进入待审核状态

#### 3.2.5 榜单页
- **榜单切换**：热门榜/新店榜/性价比榜/口味榜/环境榜
- **店铺排名列表**：展示排名、店铺名称、类型、商圈、人均价格、对应维度评分
- **点击店铺**：进入店铺详情页

#### 3.2.6 店铺详情页
- **店铺基本信息**：名称、类型、商圈、地址、人均价格
- **多维度评分**：综合评分、口味/环境/服务/性价比分项评分
- **店铺图片**：展示店铺图片
- **收藏/打卡/想去操作**：登录后可操作
- **该店铺的日记列表**：展示用户发布的关于该店铺的日记卡片

#### 3.2.7 日记详情页
- **日记全文展示**：标题、内容、发布时间、店铺名称、店铺位置
- **图片轮播**：展示日记图片
- **评分展示**：总体评分、口味/环境/服务/性价比分项评分
- **标签展示**：展示日记标签
- **作者信息**：头像、用户名、关注按钮（登录后可关注）
- **点赞操作**：登录后可点赞/取消点赞，显示点赞数
- **评论区**：展示评论列表（用户头像、用户名、评论内容、发布时间），登录后可发表评论、删除自己的评论

#### 3.2.8 个人中心页
- **用户资料展示**：头像、昵称、简介、关注数、粉丝数、获赞数
- **头像上传功能**：
  - 头像区域始终显示当前使用的头像
  - 点击头像或「编辑头像」按钮触发上传操作
  - 直接调用Supabase Storage API上传头像，不经过AuthContext updateProfile包装
  - 支持单张图片上传，格式：JPG/PNG/GIF/WEBP，最大容量5MB
  - 上传成功后头像图片保存至后端，页面立即更新显示新头像
  - 显示上传成功或失败提示
  - 上传过程中显示加载状态
- **编辑资料**：点击编辑按钮，修改昵称、简介
- **统计数据**：收藏数/已打卡数/想去数
- **标签切换**：我的日记/我的收藏/想去/已打卡
  - **我的日记**：展示用户发布的日记列表
  - **我的收藏**：展示收藏的店铺列表
  - **想去**：展示标记为想去的店铺列表
  - **已打卡**：展示标记为已打卡的店铺列表
- **发布日记按钮**：跳转发布日记页
- **关注/粉丝列表**：点击关注数或粉丝数，查看关注/粉丝列表（用户头像、用户名、关注按钮）

#### 3.2.9 登录注册页
- **登录**：输入邮箱和密码，点击登录按钮
- **注册**：输入邮箱、获取验证码、输入验证码、设置密码、确认密码，点击注册按钮
- **记住我**：勾选后保持登录状态
- **忘记密码**：点击后跳转密码重置流程

#### 3.2.10 Navbar管理员入口
- **未登录状态**：在「登录/注册」按钮旁边显示「管理后台」小链接，点击跳转至/admin/login
- **已登录普通用户**：在用户头像下拉菜单中显示「管理员登录」入口，点击跳转至/admin/login
- **已登录管理员**：在用户头像下拉菜单中显示「管理后台」链接，点击跳转至管理员后台

### 3.3 管理端页面功能

#### 3.3.1 管理员登录页（/admin/login）
- **独立登录入口**：管理员通过独立的登录页面访问后台
- **登录表单**：输入管理员邮箱和密码
- **登录验证**：验证管理员身份，仅允许角色为admin或super_admin的用户登录
- **登录失败提示**：邮箱或密码错误时提示「管理员账号或密码错误」
- **登录成功**：管理员/超级管理员登录成功后自动跳转至管理员后台，而非用户个人中心

#### 3.3.2 管理员后台
- **侧边栏导航**：数据概览/店铺管理/日记审核/用户管理/评论管理/邀请码管理（仅超级管理员可见）/系统设置
- **数据概览（仪表盘）**：展示统计卡片（待审核日记数、总店铺数、总用户数）
- **店铺管理**：
  - 店铺列表：展示店铺名称、类型、商圈、状态、操作按钮
  - 新增店铺：点击新增按钮，弹窗输入店铺信息（名称、类型、商圈、地址、描述、人均价格、图片上传），提交后创建店铺
  - 编辑店铺：点击编辑按钮，弹窗修改店铺信息
  - 删除店铺：点击删除按钮，确认后删除店铺
  - 状态管理：修改店铺状态（营业中/暂停营业/已关闭）
- **日记审核**：
  - 标签切换：待审核/已通过/已拒绝
  - 日记列表：正常加载所有待审核日记，展示日记标题、作者、店铺名称、发布时间、状态、操作按钮
  - 审核操作：
    - 点击「通过审核」按钮，可填写审核批注（可选），日记状态变为已通过（approved），日记在前台展示
    - 点击「拒绝发布」按钮，弹窗输入拒绝理由（可选），日记状态变为已拒绝（rejected），日记不在前台展示
    - 审核结果通过通知系统告知日记作者
- **用户管理**：
  - 用户列表：展示用户名、邮箱、角色、注册时间、状态
  - 禁用/启用用户：点击按钮修改用户状态
  - 修改角色：
    - 超级管理员可将用户角色修改为普通用户或管理员
    - 将用户升级为管理员时，弹出二次确认对话框，要求输入当前超级管理员账户密码进行身份验证
    - 验证通过后完成权限升级
- **评论管理**：
  - 评论列表：展示评论内容、作者、日记标题、发布时间
  - 删除评论：点击删除按钮，确认后删除评论
- **邀请码管理（仅超级管理员可见）**：
  - 邀请码列表：展示邀请码、生成者、生成时间、有效期、使用状态、使用人
  - 生成邀请码：点击生成按钮，创建新邀请码
  - 邀请码操作：
    - 复制邀请码
    - 分享邀请码
    - 查看邀请码详情（生成者、使用状态、有效期、使用人）
  - 邀请码使用规则：
    - 用户使用邀请码注册后自动获得管理员权限
    - 记录邀请码生成者、使用状态、有效期、使用人
- **搜索功能**：在各管理模块中输入关键词搜索对应内容

## 4. 业务规则与逻辑

### 4.1 用户认证规则
- 用户通过邮箱+验证码注册，验证码有效期10分钟
- 登录方式为邮箱+密码
- 用户角色分为普通用户（user）、管理员（admin）、超级管理员（super_admin）
- 超级管理员账户：3512434939@qq.com
- 未登录用户可浏览首页、探店页、探店日记页、榜单页、店铺详情页、日记详情页
- 登录后用户可发布日记、收藏店铺、关注用户、点赞日记、发表评论
- 管理员通过独立的管理员登录页（/admin/login）登录，登录验证时检查用户角色是否为admin或super_admin
- 管理员/超级管理员登录成功后自动跳转至管理员后台，而非用户个人中心
- 管理员账户与普通用户账户体系分离，管理员拥有更高权限级别

### 4.2 日记发布与审核规则
- 用户发布日记时，店铺关联为可选项，可选择关联已有店铺或不关联店铺
- 用户发布日记后，日记状态为待审核（pending）
- 管理员在日记审核页面查看待审核日记列表，列表正常加载所有待审核日记
- 管理员点击「通过审核」按钮，可填写审核批注（可选），日记状态变为已通过（approved），日记在探店日记页展示
- 管理员点击「拒绝发布」按钮，可输入拒绝理由（可选），日记状态变为已拒绝（rejected），日记不在前台展示
- 审核结果（通过或拒绝）通过通知系统告知日记作者
- 用户只能编辑或删除自己发布的日记

### 4.3 收藏与打卡规则
- 用户可对店铺进行三种操作：想去（want）、已打卡（checked）、收藏（favorite）
- 同一用户对同一店铺只能有一种状态
- 用户可在个人中心查看对应列表

### 4.4 关注规则
- 用户可关注其他用户
- 关注后可在探店日记页的「关注」标签中查看关注用户发布的日记
- 用户可在个人中心查看关注列表和粉丝列表

### 4.5 点赞与评论规则
- 用户可对日记点赞，点赞后可取消点赞
- 用户可对日记发表评论
- 用户可删除自己发表的评论
- 管理员可删除任何评论

### 4.6 评分计算规则
- 店铺的综合评分为所有日记对该店铺的总体评分的平均值
- 店铺的口味/环境/服务/性价比分项评分为所有日记对应分项评分的平均值
- 评分保留一位小数

### 4.7 榜单生成规则
- **热门榜**：按店铺综合评分降序排列
- **新店榜**：按店铺创建时间降序排列
- **性价比榜**：按（综合评分 / 人均价格）降序排列
- **口味榜**：按店铺口味评分降序排列
- **环境榜**：按店铺环境评分降序排列

### 4.8 搜索规则
- 首页搜索框：搜索店铺名称或日记标题/内容
- 探店页搜索框：搜索店铺名称
- 管理员后台搜索：在各管理模块中搜索对应内容（店铺名称、用户名、日记标题等）

### 4.9 图片上传规则
- 用户头像上传：直接调用Supabase Storage API，存储在avatars桶，支持JPG/PNG/GIF/WEBP格式，最大容量5MB，超过1MB自动压缩为WEBP格式，分辨率1080p，质量0.8，上传成功后头像图片保存至后端并更新用户资料
- 日记图片上传：存储在diary-images桶，超过1MB自动压缩为WEBP格式，分辨率1080p，质量0.8
- 店铺图片上传：存储在shop-images桶，超过1MB自动压缩为WEBP格式，分辨率1080p，质量0.8

### 4.10 权限管理规则
- 超级管理员可将普通用户升级为管理员
- 升级操作需二次确认：弹出对话框要求输入当前超级管理员账户密码
- 密码验证通过后完成权限升级
- 超级管理员可生成邀请码，邀请码可复制/分享
- 用户使用邀请码注册后自动获得管理员权限
- 邀请码记录生成者、使用状态、有效期、使用人

### 4.11 视觉统一规则
- 全站统一色彩主题：暖橙色系为主色调
- 统一字体、按钮样式、表单元素、图标风格
- 导航栏/侧边栏/页脚在所有页面保持一致
- 统一栅格系统和间距规范

## 5. 数据模型

### 5.1 profiles表
- id (uuid, 外键关联auth.users)
- username (text, 用户名)
- email (text, 邮箱)
- avatar_url (text, 头像URL)
- bio (text, 简介)
- role (enum: user/admin/super_admin, 角色)
- followers_count (int, 粉丝数)
- following_count (int, 关注数)
- created_at (timestamp, 创建时间)

### 5.2 shops表
- id (uuid, 主键)
- name (text, 店铺名称)
- type (text, 菜系类型)
- area (text, 商圈)
- address (text, 地址)
- description (text, 描述)
- images (jsonb, 图片数组)
- price_per_person (int, 人均价格)
- rating_avg (float, 综合评分)
- rating_count (int, 评分数量)
- rating_taste (float, 口味评分)
- rating_env (float, 环境评分)
- rating_service (float, 服务评分)
- rating_value (float, 性价比评分)
- status (enum: active/pending/closed, 状态)
- created_by (uuid, 创建者ID)
- created_at (timestamp, 创建时间)

### 5.3 diaries表
- id (uuid, 主键)
- user_id (uuid, 用户ID)
- shop_id (uuid, 店铺ID，可为空)
- title (text, 标题)
- content (text, 内容)
- images (jsonb, 图片数组)
- rating_overall (float, 总体评分)
- rating_taste (float, 口味评分)
- rating_env (float, 环境评分)
- rating_service (float, 服务评分)
- rating_value (float, 性价比评分)
- tags (text[], 标签数组)
- likes_count (int, 点赞数)
- comments_count (int, 评论数)
- status (enum: pending/approved/rejected, 状态)
- reject_reason (text, 拒绝理由，可为空)
- review_note (text, 审核批注，可为空)
- created_at (timestamp, 创建时间)

### 5.4 diary_likes表
- user_id (uuid, 用户ID)
- diary_id (uuid, 日记ID)
- created_at (timestamp, 创建时间)
- 联合主键：(user_id, diary_id)

### 5.5 diary_comments表
- id (uuid, 主键)
- diary_id (uuid, 日记ID)
- user_id (uuid, 用户ID)

## 7. 验收标准

1. 用户访问首页，点击「火锅」分类，跳转探店页并自动筛选火锅类店铺
2. 用户在探店页选择「春熙路」商圈，列表展示春熙路商圈的店铺
3. 用户点击某店铺卡片，进入店铺详情页，查看店铺信息和日记列表
4. 用户点击注册，输入邮箱、获取验证码、输入验证码、设置密码，完成注册并自动登录
5. 用户登录后进入个人中心，点击头像或「编辑头像」按钮，上传新头像（JPG格式，3MB），直接调用Supabase Storage API上传，上传成功后头像图片保存至后端，页面立即显示新头像并提示「头像更新成功」
6. 用户点击「发布日记」，选择「不关联店铺」，填写评分、标题、内容、上传图片、添加标签，提交后日记进入待审核状态
7. 未登录用户在Navbar点击「管理后台」小链接，跳转至/admin/login管理员登录页
8. 超级管理员（3512434939@qq.com）通过/admin/login登录页输入邮箱和密码登录，自动跳转至管理员后台（而非个人中心），进入管理员仪表盘，查看待审核日记数、总店铺数、总用户数
9. 超级管理员进入日记审核页面，正常加载所有待审核日记列表，点击「通过审核」按钮并填写审核批注，日记状态变为已通过，审核结果通过通知系统告知日记作者
10. 用户在探店日记页查看已通过的日记，点击日记卡片进入日记详情页
11. 用户在日记详情页点击点赞按钮，点赞数+1，再次点击取消点赞
12. 用户在日记详情页发表评论，评论显示在评论区
13. 用户在店铺详情页点击「收藏」按钮，店铺被添加到个人中心的「我的收藏」列表
14. 超级管理员在用户管理页面将某普通用户升级为管理员，弹出二次确认对话框，输入当前账户密码验证通过后完成权限升级
15. 超级管理员在邀请码管理页面生成新邀请码，复制邀请码并分享给新用户
16. 新用户使用邀请码注册，注册成功后自动获得管理员权限
17. 已登录管理员在Navbar用户头像下拉菜单中点击「管理后台」链接，直接进入管理员后台
